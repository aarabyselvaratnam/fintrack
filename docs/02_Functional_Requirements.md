Functional Requirements

1. Authentication

The system shall:

- Allow users to register using their name, email, and password.
- Allow users to log in using email and password.
- Encrypt user passwords before storing them.
- Authenticate users using JWT.
- Allow users to log out securely.
- Prevent unauthorized users from accessing protected pages.


2. Dashboard

The system shall:

- Display the user's current balance.
- Display total income.
- Display total expenses.
- Show recent transactions.
- Display budget progress.
- Show spending summaries using charts.


3. Income Management

The system shall:

- Allow users to add income.
- Edit income records.
- Delete income records.
- Categorize income.
- View income history.


4. Expense Management

The system shall:

- Allow users to add expenses.
- Edit expenses.
- Delete expenses.
- Categorize expenses.
- Record transaction dates.
- Add notes for expenses.


5. Transaction Management

The system shall:

- Display all transactions.
- Search transactions.
- Filter transactions by:
  - Category
  - Date
  - Income
  - Expense
- Sort transactions.
- View transaction details.


6. Categories

The system shall:

- Provide default categories.
- Allow users to create custom categories.
- Edit categories.
- Delete custom categories.


7. Budget Management

The system shall:

- Allow users to create monthly budgets.
- Set budget limits for categories.
- Track remaining budget.
- Display budget progress.
- Warn users when budgets are exceeded.


8. Reports & Analytics

The system shall:

- Generate monthly reports.
- Generate yearly reports.
- Show income vs expense comparison.
- Display spending by category.
- Show financial trends using charts.


9. User Profile

The system shall:

- Allow users to update personal information.
- Change passwords.
- Upload profile pictures.
- Manage account settings.

10. Notifications

The system shall:

- Notify users when budgets are exceeded.
- Notify users about monthly spending summaries.
- Display important financial reminders.

11. Security

The system shall:

- Protect private user data.
- Validate all user inputs.
- Restrict access to authorized users only.
- Secure API endpoints.
